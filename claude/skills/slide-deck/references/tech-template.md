# 技術説明テンプレート

## 用途
エンジニア・CTO向けの技術詳細説明、アーキテクチャ提案、技術選定の提案など。
product社向け説明、技術パートナーとの協議、採用判断などに使用。

---

## スライド構成（標準8〜10枚）

### 1. 表紙
```
[技術名 / システム名] 概要説明

[サブタイトル: 何を説明するか一言で]

[対象者]: [エンジニア / CTO / 技術担当]
[日付] | [発表者名]
```

### 2. TL;DR（Too Long; Didn't Read）
```
■ 要約（3点）

1. [技術的ポイント1 + 数字]
2. [技術的ポイント2 + 数字]
3. [推奨アクション]

→ 詳細は以降のスライドで説明
```

### 3. 技術的課題（Problem）
```
■ 現状の技術課題

・[課題2]: [影響] (例: スケーラビリティの限界)
・[課題3]: [影響] (例: 保守コスト月¥XXX万)
```

```
・言語: [技術名]

```

## ブロックチェーン・Web3説明のコツ

### 難しい用語を翻訳する
| 専門用語 | わかりやすい言葉 |
|---------|----------------|
| ブロックチェーン | 改ざんできない分散型データベース |
| スマートコントラクト | 条件が満たされたら自動実行するプログラム |
| ウォレット | 秘密鍵管理ソフト（銀行口座のキーホルダー）|
| ガス代 | トランザクション手数料 |
| ノード | ネットワーク参加コンピュータ |
| コンセンサス | 全ノードが合意する仕組み |

### product社への説明でよく使う構成
1. なぜブロックチェーンが必要か（vs 従来DB）
2. 選定チェーン・プロトコルの根拠
3. セキュリティモデル
4. ガバナンス・アップグレード戦略
5. コスト構造（ガス代の試算）
