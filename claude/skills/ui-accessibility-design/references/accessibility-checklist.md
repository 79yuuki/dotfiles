# Accessibility Checklist — WCAG AAA準拠

レビュー・設計時に使うチェックリスト。必須（AAA）と推奨（追加対応）に分類。

---

## 視覚（Perceivable）

### コントラスト比 [必須]
- [ ] 通常テキスト（18px未満 / 14px未満Bold）: **7:1以上**
- [ ] 大テキスト（18px以上 / 14px以上Bold）: **4.5:1以上**
- [ ] UIコンポーネント・グラフィック: **3:1以上**
- [ ] プレースホルダーテキスト: 4.5:1以上（AAA要件）
- [ ] フォーカスリング: 背景に対して3:1以上

**ツール**: [Colour Contrast Analyser](https://www.tpgi.com/color-contrast-checker/) / Chrome DevTools

### テキスト・フォント [必須]
- [ ] 本文フォントサイズ: **16px以上**（モバイルも同様）
- [ ] 行間（line-height）: 1.5以上
- [ ] 段落間のスペース: フォントサイズの1.5倍以上
- [ ] 文字間隔（letter-spacing）調整後でも読めるか
- [ ] 200%ズームで横スクロールが発生しないか

### 色依存 [必須]
- [ ] 色だけで情報を伝えていないか（アイコン・テキスト・パターンで補足）
- [ ] エラー状態: 赤色のみでなく、アイコン+テキストでも表現
- [ ] グラフ・チャート: 色盲ユーザーも識別できるか（パターン・ラベル追加）

### 画像・メディア [必須]
- [ ] 情報を持つ画像にalt属性あり（内容を説明、「画像」という単語不要）
- [ ] 装飾画像はalt=""（空文字）
- [ ] 動画に字幕あり
- [ ] 音声コンテンツにトランスクリプトあり

---

## 操作（Operable）

### キーボード操作 [必須]
- [ ] Tab/Shift+Tabで全インタラクティブ要素に到達できる
- [ ] フォーカスの順序が論理的（左上→右下の読み順）
- [ ] フォーカスが画面外に消えない（フォーカストラップの適切な管理）
- [ ] Escキーでモーダル・ドロップダウンが閉じる
- [ ] Enterキーでボタン・リンクが実行できる
- [ ] 矢印キーでラジオボタン・タブが操作できる

### フォーカス表示 [必須]
- [ ] フォーカスリングが明確に見える（デフォルトを消していない）
- [ ] カスタムフォーカスリングはコントラスト3:1以上
- [ ] アウトラインサイズ: 2px以上

### タッチターゲット [必須]
- [ ] タッチターゲット: **44×44px以上**（モバイル）
- [ ] 隣接するターゲット間の間隔: 8px以上
- [ ] 小さいアイコンボタンには十分なパディングを追加

### 時間制限 [必須]
- [ ] 時間制限のあるコンテンツは延長/無効化できる
- [ ] 自動更新するコンテンツは停止できる
- [ ] セッションタイムアウト前に警告を表示（少なくとも20秒前）

### 動き・アニメーション [必須]
- [ ] `prefers-reduced-motion` に対応（アニメーション無効化）
- [ ] 点滅・フラッシュ: 1秒間に3回以下 or 完全回避
- [ ] スクロールに追従する要素（Parallax）は無効化オプションあり

---

## 理解しやすさ（Understandable）

### フォーム [必須]
- [ ] ラベルは常に表示（プレースホルダーのみ禁止）
- [ ] エラーメッセージ: 何が問題で、どう修正すればいいかを明示
- [ ] エラーメッセージはフィールドに紐付いている（aria-describedby）
- [ ] 必須フィールドは視覚的+テキストで示す（* のみ禁止）
- [ ] 送信前に入力内容の確認・修正機会がある（重要な操作）
- [ ] autocomplete属性を設定（名前・メール・住所等）

### コピーライティング [必須]
- [ ] ページタイトルでページ内容がわかる
- [ ] リンクテキストが「こちら」「詳しく」のみでない（何へのリンクか明確）
- [ ] ボタンラベルは動詞で始まる（送信 / 削除 / 保存）
- [ ] エラーメッセージにHTTPステータスコードのみでない

### 予測可能性 [必須]
- [ ] フォーカス移動だけで意図しないページ変化が起きない
- [ ] 同じUIは同じ動作をする（一貫性）
- [ ] ナビゲーションはページ間で一貫している

### 認知負荷 [推奨]
- [ ] 1画面のメインアクションは1つ
- [ ] 長いフォームはステップ表示（現在位置がわかる）
- [ ] 複雑な操作には説明文・ツールチップを追加

---

## スクリーンリーダー対応（Robust）

### セマンティクス [必須]
- [ ] 見出しの階層が正しい（h1→h2→h3、スキップなし）
- [ ] ランドマーク要素を使用（header, nav, main, footer, aside）
- [ ] ボタンは`<button>`、リンクは`<a href>`（divクリックイベントは禁止）
- [ ] テーブルにcaption・th（scope属性付き）がある

### ARIA [必須]
- [ ] aria-labelまたはaria-labelledbyで要素の目的を説明（ラベルがない場合）
- [ ] 動的コンテンツにaria-live属性（通知・エラーの読み上げ）
- [ ] モーダルにrole="dialog"・aria-modal="true"
- [ ] アイコンのみのボタンにaria-label
- [ ] aria-hiddenで装飾要素をスクリーンリーダーから隠す

### フォーム（ARIA） [必須]
- [ ] inputにid、labelにfor（またはaria-labelledby）
- [ ] エラーにrole="alert"またはaria-live="polite"
- [ ] グループ化されたフォームにfieldset+legend

## 優先度マトリクス

| 影響度 | 対応難易度 | 優先度 |
|---|---|---|
| コントラスト比 | 低 | 🔴 最優先 |
| エラーメッセージ改善 | 低 | 🔴 最優先 |
| キーボード操作 | 中 | 🟠 高 |
| フォーカス表示 | 低 | 🟠 高 |
| ARIA基本実装 | 中 | 🟠 高 |
| タッチターゲット | 低 | 🟡 中 |
| reduced-motion | 低 | 🟡 中 |
| スクリーンリーダー完全対応 | 高 | 🟢 余裕があれば |
