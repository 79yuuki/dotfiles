## Weekly review
- new opportunities
- stuck opportunities
- proposals sent
- follow-ups due
- lost/dormant reasons
- assets missing repeatedly

## Distribution / relationship ownership check
For each offer, partnership, x402/JPYC/DeFi opportunity, bot strategy, or creator business motion, record:
- distribution owner: who owns the first touch and repeat access?
- relationship depth: one-off transaction / subscribed user / wallet or account relationship / embedded workflow
- default path: what UI, wallet, platform, API, or social channel routes the user by default?
- capture mechanism: fee, spread, rev-share, data, audience, renewal, or strategic option value
- dependency risk: which upstream distribution owner could copy, tax, block, or replace us?
- next action to improve capture: embed, partnership, owned list, wallet/front-end placement, referral loop, or account expansion

Weekly review prompt: “Which active opportunity creates value but fails to capture it because another party owns distribution?”

## Common failure modes
- one tracker per project
- no stale limit
- no asset inventory
- follow-up depends on memory
