# Decision Trace Template

Use this when proposal quality depends on preserving why a choice was made, not only what stage changed.

## Minimal objects
- account / opportunity
- decision id
- date
- owner
- participants
- linked proposal / meeting / thread

## Required fields
- context
- default proposal
- chosen action
- override? (yes/no)
- override reason
- expected upside
- expected downside
- next review date
- actual outcome
- lesson / policy update

## Common decision types
- pricing exception
- custom scope approval
- rebate or discount approval
- payment terms override
- target account prioritization
- whether to pursue / pause / drop a deal

## Good entry example
- Context: Enterprise lead wants custom onboarding and asks for a discount.
- Default proposal: standard setup fee + monthly retainer.
- Chosen action: waive setup fee, keep retainer.
- Override reason: logo value + likely expansion path.
- Expected upside: faster close, strong case study.
- Expected downside: precedent risk.
- Next review date: 30 days after go-live.
- Actual outcome: closed quickly but asked for more custom work.
- Lesson: waive setup fee only when expansion path is written down.

## Weekly review prompts
- Which overrides paid off?
- Which overrides became precedent debt?
- Which recurring questions need policy?
- Which lost deals need a clearer rationale tag?
