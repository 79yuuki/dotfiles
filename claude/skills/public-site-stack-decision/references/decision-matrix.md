# Public Site Stack Decision Matrix

## Score each option on
- editor ease
- structured content support
- localization
- performance
- design freedom
- security/update burden
- preview workflow
- hosting/deploy simplicity
- observability
- total maintenance cost

## Strong fit examples

### WordPress
- non-technical editors need control
- plugin ecosystem is genuinely required
- maintenance owner exists

### Static site
- low content churn
- high performance
- developer-owned publishing

### Headless CMS + frontend
- multiple content surfaces share structured data
- editorial workflow matters
- custom frontend needed

### Docs platform / docs framework
- versioned docs
- API reference
- search + docs IA matter more than page-builder freedom
