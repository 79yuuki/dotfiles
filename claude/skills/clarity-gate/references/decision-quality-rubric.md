# Decision Quality Rubric for Strategy Docs

戦略ドキュメント、decision memo、方針判断メモ向けの軽量ルーブリック。

狙いは**「読みやすい」だけで終わらせず、判断の前提と見直し条件まで残すこと。**

## いつ使うか

- 新しい戦略を提案する時
- ラウンド、提携、GTM、採用、公開方針などの判断を残す時
- 「今回は何を選ぶか」を決める文書を書く時
- 後から見返して「なぜそう決めたか」を再現したい時

## 1ページ基本フォーマット

以下の8項目があれば十分。

1. **Decision**
   - 今回決めることを1文で書く
   - 例: 「x402のenterprise向けLPは、payment rail訴求ではなく trust / approval 主語で再構成する」

2. **Why now**
   - なぜ今この判断が必要か
   - 放置コスト、締切、機会損失を書く

3. **Options compared**
   - 比較した選択肢を2〜4個並べる
   - **現状維持 / 何もしない** を必ず1つ入れる

4. **Assumptions**
   - この判断が成り立つ前提を書く
   - 市場仮説、顧客理解、工数見積もり、依存関係など

5. **Recommendation**
   - 今回の推奨案を1つに絞る
   - その理由を3行以内で書く

6. **Disconfirming signals**
   - この判断が外れているとわかるサインを書く
   - 例: CVが改善しない、商談で刺さらない、制作コストが見合わない

7. **Revisit date / owner**
   - いつ見直すか
   - 誰が見直すか

8. **Immediate next step**
   - 次に開くIssue / Doc / 実験を具体的に書く
   - 5〜15分で着手できる粒度にする

## 赤信号チェック

以下のどれかが欠けたら、判断文書としては弱い。

- **Decisionが曖昧**
  - 読み終えても「結局なにを決めたいのか」がわからない
- **Optionsが1個しかない**
  - 推奨案しかなく、比較になっていない
- **Assumptionsが隠れている**
  - 前提が本文の行間に埋まっていて、崩れた時に見直せない
- **反証条件がない**
  - 失敗の見え方が定義されていない
- **見直し日がない**
  - その場の勢いで決めっぱなしになる
- **次アクションが抽象的**
  - 「検討する」「進める」で終わっている

## 3分レビュー用ミニ質問

文書を読んで、次の質問に全部答えられるか確認する。

1. 今回なにを決めるの？
2. なんで今決めるの？
3. 他にどんな選択肢があったの？
4. その判断は何を前提にしてるの？
5. どんな結果が出たら見直すの？
6. 次に誰が何をやるの？

1つでも詰まるなら、文書に追記する。

## 出力テンプレート

```md
## Decision
[1文で書く]

## Why now
- [なぜ今か]
- [締切 / 放置コスト]

## Options compared
- Option A: [案]
- Option B: [案]
- Option C: Do nothing / 現状維持

## Assumptions
- [前提1]
- [前提2]

## Recommendation
[今回は何を採用するか]

## Disconfirming signals
- [この条件なら見直す]
- [この結果なら撤退 / 方向転換]

## Revisit date / owner
- Date: [YYYY-MM-DD]
- Owner: [name]

## Immediate next step
- [次に開くIssue / Doc / 実験]
```

## 他スキルとの使い分け

- **clarity-gate**: 読みやすさ + 判断文書として必要な骨格をチェックする
- **winning-proposal**: 提案が通る構造を作る
- **sales-gtm-os**: 判断ログや営業運用に載せる
- **fact-check-gate**: 数字・社名・日付・URLなどの事実を確認する

つまり、**このルーブリックは「判断の型」を軽く揃えるための下地**として使う。
