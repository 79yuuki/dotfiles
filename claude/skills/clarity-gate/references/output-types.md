# アウトプット種別と適用基準

## 種別一覧

### 🔴 最厳格（Strict）

**対象:** LP、広告コピー、SNS投稿、プレスリリース

- 不特定多数が読む → 7人に1人の境界知能の人も含まれる
- 離脱/誤解が直接的なビジネスインパクト
- **全チェック項目を適用**
- 専門用語は原則禁止（使う場合は必ず括弧内に平易な説明）
- 一文30文字以内を目標
- CTAは必ず具体的な動詞で始める

### 🟠 厳格（Firm）

**対象:** ユーザードキュメント、README、使い方ガイド、FAQ、ブログ記事

- 操作や理解に直結 → 誤解はサポートコスト増・ユーザー離脱
- **A（構造）B（言葉）C（流れ）E（行動）を重点チェック**
- 専門用語は初出のみ説明
- ステップバイステップの手順は必須
- スクリーンショット/図の活用を推奨

### 🟡 標準（Standard）

**対象:** 提案書、見積書、企画書、戦略ドキュメント、decision memo、社外メール

- 読者は限定的だが、理解と承認が必要
- **B（言葉）C（流れ）E（行動）を重点チェック**
- 専門用語は読者の知識レベルに合わせて判断
- 結論先行（エグゼクティブサマリー）必須
- 金額・条件・期限は明確に
- 戦略 / 判断メモでは `decision-quality-rubric.md` の add-on も実行する

### 🟢 基本（Basic）

**対象:** team chat報告、社内ドキュメント、技術ドキュメント、議事録

- 社内読者が主 → 共通知識を前提にできる
- **A（構造）C（流れ）を重点チェック**
- 専門用語は社内で共有されていれば説明不要
- ただし構造化（見出し・箇条書き）は常に必須
- 長文の報告には「TL;DR」をつける

## 種別の自動判定ルール

以下のキーワード/コンテキストで種別を自動判定:

| キーワード | 種別 |
|-----------|------|
| LP, ランディングページ, 広告, ad, コピー | 🔴 |
| ツイート, X投稿, SNS, Instagram | 🔴 |
| プレスリリース, PR | 🔴 |
| README, ガイド, マニュアル, FAQ, ヘルプ | 🟠 |
| ブログ, 記事, post | 🟠 |
| 提案書, 見積, 企画書, proposal | 🟡 |
| 戦略, strategy, decision memo, 意思決定, 方針, go-to-market | 🟡 |
| メール（社外） | 🟡 |
| team chat, 報告, 議事録, メモ | 🟢 |
| 技術ドキュメント, API doc, spec | 🟢 |

判定に迷う場合は一段階厳しい方を適用する。
