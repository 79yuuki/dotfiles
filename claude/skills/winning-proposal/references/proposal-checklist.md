# 提案書 構造チェックリスト

提案書の設計完了後、このリストで最終確認する。

## 6要素チェック（全て ✅ で合格）

- [ ] **なぜ今:** 相手の経営文脈で「今」の理由を語れているか
- [ ] **やらないコスト:** 現状維持のリスク・コストを具体的に示しているか
- [ ] **Before/After:** 変化を数字（またはシナリオ）で見せているか
- [ ] **なぜ自社:** 相手の重視軸で差別化を語れているか
- [ ] **ROI/回収:** 総費用・期待効果・回収期間が明示されているか
- [ ] **Next Step:** 具体的な次のアクション＋期日があるか

## NGパターン回避（全て ✅ で合格）

- [ ] **自社視点排除:** 冒頭が自社紹介（シェア、受賞歴、機能一覧）になっていない
- [ ] **経営語翻訳:** 現場の改善 → 事業インパクトまで繋げている
- [ ] **レイヤー対応:** 決裁レイヤーに応じた粒度・観点になっている

## 担当者武装化チェック

- [ ] 担当者がこの資料だけで上司に説明できるか
- [ ] 「この導入を推進した担当者の評価が上がる根拠」が含まれているか
- [ ] エグゼクティブサマリー1枚が用意されているか（役員が3分で把握可能）
- [ ] 想定Q&A（上位者から聞かれそうな質問への回答）を添付しているか

## 決算期タイミングチェック（3月決算企業の場合）

| 時期 | 担当者の状況 | 提案のポイント |
|------|-----------|-------------|
| 8-9月 | 下期戦略見直し、翌期投資計画の議論開始 | 概算見積もり提示、足の長い案件はここで仕込む |
| 10-12月 | 来期予算の原案作成 | 正式提案＋ROI資料、担当者の武器を渡す |
| 1-2月 | 予算の社内調整・承認プロセス | エグゼクティブサマリー、想定Q&A強化 |
| 3月 | 最終調整・駆け込み | 「今期中に決裁」のための緊急性訴求 |

## 提案書 → スライド変換ガイド

構造チェック通過後、出力フォーマットを選択:

1. **Marpスライド** → `slide-deck` スキルのSCQA構造に6要素をマッピング
   - S(Situation) ← 要素1「なぜ今」
   - C(Complication) ← 要素2「やらないコスト」
   - Q(Question) ← 暗黙（どうすればいいか）
   - A(Answer) ← 要素3-6（Before/After、差別化、ROI、Next）
2. **ビジュアルスライド** → `visual-slides` で画像生成
3. **Word/PDF** → テンプレートに流し込み
