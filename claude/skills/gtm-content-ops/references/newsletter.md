# ニュースレター運用

## 「ゼロから書く」から「既存コンテンツから展開」へ

ニュースレターを毎回ゼロから書くのは続かない。
日々のSNS投稿・記事から自動的にニュースレター素材を作る仕組みにする。

## Beehiivの活用

### なぜBeehiivか
- 無料プランで始められる（2,500購読者まで）
- SEO対応のWebページ自動生成
- リファラルプログラム内蔵
- カスタムドメイン対応
- 分析ダッシュボードが充実

### 代替サービス
| サービス | 特徴 |
|---|---|
| Substack | ライター向け、有料購読に強い |
| ConvertKit | クリエイター向け、自動化に強い |
| Mailchimp | 老舗、EC連携に強い |
| Buttondown | シンプル、Markdown対応 |

## 配信リズム

### 週次配信（推奨）
- **配信日**: 火曜 or 水曜の朝（開封率が高い）
- **準備**: 金曜に素材ピックアップ → 月曜に構成 → 配信
- **構成**:
  1. 今週のハイライト（1-2本、最も反応が良かったコンテンツ）
  2. ニュースまとめ（3-5本）
  3. 今週の学び/裏側（個人の声）
  4. 来週の予告

### 月次配信（ライト版）
- 月間ベスト記事
- 月間ニュースまとめ
- 来月の展望

## コンテンツキュレーション自動化

### 素材収集の仕組み

```
日々のフロー:
  SNS投稿 → エンゲージメント計測 → 上位をNL候補にタグ付け
  記事公開 → PV計測 → 上位をNL候補にタグ付け
  
週次の処理:
  NL候補一覧 → 選定（5-8本） → NL構成に配置 → 文章整形 → 配信
```

### 自動化できる部分
- エンゲージメント上位の自動抽出
- NLテンプレートへの自動配置
- 配信スケジュールの自動設定

### 手動が必要な部分
- 「今週の学び」のパーソナルな一言
- コンテンツの並び順の最終調整
- 特別なお知らせの追加

## KPI

| 指標 | 目安（スタート時） | 成長期 |
|---|---|---|
| 開封率 | 40-50% | 30-40% |
| クリック率 | 5-10% | 3-7% |
| 購読者数 | - | 月+10-20% |
| 解除率 | < 1% | < 0.5% |

## ニュースレター → 他コンテンツ展開

- **NL → SNS**: 「今週のNLで○○について書きました」と各プラットフォームで告知
- **NL → ブログ**: NL限定コンテンツの一部をブログに転載（アーカイブ化）
- **NL → PR**: 購読者数のマイルストーン達成をプレスリリースに
