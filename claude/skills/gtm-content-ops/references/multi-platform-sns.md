# マルチプラットフォームSNS運用

## 1トピック × Nプラットフォーム方式

1つのネタに対して、プラットフォームごとに最適化したテキストを生成して同時投稿する。
85トピック × 5プラットフォーム = 425投稿/月 の実績あり。

## プラットフォーム別整形ルール

### X（Twitter）
- 280文字制限（日本語140文字目安）
- スレッド展開で深掘り可
- ハッシュタグ: 1-2個、関連性の高いもの
- 画像/動画で停止率UP
- リプライでの補足情報追加

### LinkedIn
- ビジネス文脈に寄せる
- 段落分けで読みやすく（改行多め）
- 個人の学び・意見を入れる
- 3000文字まで
- 冒頭3行で引きを作る（「続きを読む」の前）

### Instagram
- ビジュアル必須（カルーセル推奨）
- キャプションは長文OK
- ハッシュタグ: 10-15個
- ストーリーズでの補足展開
- リール: 60-90秒の要約動画

### Facebook
- コミュニティ・グループ展開
- 長文OK
- リンク付き投稿のリーチ低下に注意
- コメント欄にリンクを置く戦略

### note / ブログ
- 長文の深掘り記事
- SEOを意識した構成
- SNS投稿を膨らませた形

## バッチ処理フロー

```
1. トピック選定（朝のニュースチェック or 夜の日報）
2. コアメッセージを1文で定義
3. 各プラットフォーム用に変換:
   - X: 短縮 + フック
   - LinkedIn: ビジネス文脈 + 意見
   - Instagram: ビジュアル + キャプション
   - Facebook: コミュニティ向け + 議論誘発
   - note: 深掘り（必要な場合のみ）
4. 一括投稿（ツール or 手動）
5. エンゲージメント計測（翌日確認）
```

## 投稿ツール候補

| ツール | 対応プラットフォーム | 特徴 |
|---|---|---|
| Buffer | X, LinkedIn, Instagram, Facebook, Pinterest | スケジュール投稿、分析 |
| Hootsuite | 主要SNS全般 | エンタープライズ向け |
| SocialBee | X, LinkedIn, Instagram, Facebook | カテゴリ別ローテーション |
| カスタムスクリプト | 任意 | API直叩き、完全自動化 |
| agent runtime + sessions_spawn | X（browser操作） | Muser環境で直接投稿可 |

## エンゲージメント計測

### 追跡指標（プラットフォーム共通）
- インプレッション / リーチ
- エンゲージメント率（いいね + RT + コメント / インプレッション）
- クリック率（リンク付きの場合）
- フォロワー増減

### 判断基準
- エンゲージメント率 > 3%: 深掘り記事化の候補
- インプレッション突出: プレスリリースのネタ候補
- コメント多数: 次のコンテンツのネタ

## 注意点

- **各プラットフォームのアルゴリズム変更に注意**: 特にXとInstagramは頻繁に変わる
- **コピペ投稿は避ける**: 必ずプラットフォームに合わせて整形する
- **投稿頻度の適正化**: 多すぎるとフォロワー離脱。1日2-4投稿が目安
- **自分の声を入れる**: AIで生成したままだと「表面的」になりがち
