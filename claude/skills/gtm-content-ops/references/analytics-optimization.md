# 分析と最適化

## 「量は出せた。質の最適化はこれから」フェーズ

量産体制を構築した後の課題:
1. コンテンツの品質がまだ低い部分がある（自分らしさが足りない、表面的）
2. 何が伸びるかの分析がまだ不十分
3. 「読者の役に立っているか」の検証ができていない

## 分析フレームワーク

### レイヤー1: 数値分析

| プラットフォーム | 主要指標 | 取得方法 |
|---|---|---|
| X | インプレッション、エンゲージメント率、プロフィールクリック | Analytics API / ダッシュボード |
| LinkedIn | インプレッション、エンゲージメント、フォロワー増減 | Analytics |
| ブログ/サイト | PV、滞在時間、直帰率、検索流入キーワード | GA4 / Search Console |
| NL | 開封率、クリック率、解除率 | Beehiiv Analytics |
| PR TIMES | PV、メディア掲載数、転載数 | PR TIMES ダッシュボード |

### レイヤー2: コンテンツ分析

**勝ちパターンの特定:**
```
1. 過去30日のコンテンツを全件リスト化
2. エンゲージメント順にソート
3. 上位20%の共通点を抽出:
   - トピック（何について書いたか）
   - フォーマット（How-to / ニュース / 意見 / 事例）
   - 投稿時間帯
   - テキスト長
   - 感情の強さ（熱量の有無）
4. 下位20%の共通点も抽出（避けるべきパターン）
```

**コンテンツタイプ別パフォーマンス:**
| タイプ | 期待エンゲージメント | 注意点 |
|---|---|---|
| ニュース速報 | 高（鮮度が命） | 差別化しにくい |
| 独自意見/考察 | 中-高 | 自分の声が必要 |
| ハウツー/ガイド | 中（長期的にSEO効果） | 更新コストあり |
| 裏側/開発ログ | 中-高 | 共感を得やすい |
| 事例紹介 | 中 | 許可が必要 |
| まとめ/リスト | 高（保存されやすい） | 付加価値が必要 |

### レイヤー3: 「自分らしさ」の担保

AIで量産すると「表面的」「どこかで見た感じ」になりがち。

**差別化ポイントの注入:**
- 独自の体験談（「実際に○○を使ってみたら」）
- 明確な意見（「個人的には△△だと思う」）
- 業界の裏話（「実はこの背景には□□がある」）
- 比喩や言い回しの癖（文体の個性）

**チェックリスト（記事公開前）:**
- [ ] 自分にしか書けない一文が入っているか
- [ ] 読者が「この人の意見が聞きたい」と思えるか
- [ ] AIが生成したままの表現が残っていないか
- [ ] 具体的な数字やデータがあるか

## 最適化サイクル

```
Week 1-2: 計測期間（変更せずデータ収集）
Week 3: 分析（勝ちパターン/負けパターン特定）
Week 4: 改善実行（構成/トピック/投稿時間を調整）
→ 次の計測期間へ
```

## ツール

- **Google Analytics 4**: サイトトラフィック分析
- **Search Console**: 検索パフォーマンス
- **各SNSのネイティブ分析**: プラットフォーム固有の指標
- **Beehiiv Analytics**: NL分析
- **スプレッドシート/Notion**: 横断的なコンテンツパフォーマンスDB
