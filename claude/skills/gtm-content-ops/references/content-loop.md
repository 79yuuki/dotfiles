# コンテンツループ設計パターン

## 基本ループ構造

一人法人のコンテンツ運用は「ループ」で回す。直線的な生産ではなく、循環型。

### インプット → アウトプット変換マトリクス

| インプット | → SNS投稿 | → 記事 | → NL | → PR | → ネタ |
|---|---|---|---|---|---|
| AIニュース | ◎ 即時投稿 | ○ 解説記事化 | ○ 週次まとめ | △ トレンド文脈 | ○ |
| 日報（音声入力） | ◎ 日報展開 | ○ ノウハウ化 | ○ 裏側共有 | × | ◎ |
| 開発ログ | ◎ 進捗共有 | ◎ 技術記事化 | ○ アップデート | ◎ リリース | ◎ |
| 顧客の声 | ○ 事例紹介 | ◎ ケーススタディ | ○ 成果報告 | ◎ 導入事例 | ○ |
| イベント登壇 | ◎ 実況/レポ | ◎ 登壇レポート | ○ ハイライト | ◎ 登壇PR | ○ |
| 過去コンテンツ | ◎ リマインド | ◎ リライト | ○ ベスト版 | × | × |

### ループ接続パターン

```
パターン1: ニュース駆動（朝ルーティン）
  ニュース収集 → 5プラットフォームSNS投稿 → 反応良いもの → 解説記事化
  
パターン2: 日報駆動（夜ルーティン）
  音声日報入力 → プラットフォーム別整形 → SNS投稿 → 週次NL素材
  
パターン3: 開発駆動（随時）
  機能リリース → プレスリリース + SNS + 記事 → NL特集
  
パターン4: 棚卸し駆動（週次/月次）
  過去の未プレス化イベント → PR化 → SNS展開 → 記事化
  
パターン5: メタ駆動（自己参照）
  仕組みの開発・改善そのもの → 記事「こうやって回してます」→ SNS
```

## 運用リズム設計

### 日次（朝: 15-30分）
1. ニュースソース巡回（RSS/X/業界メディア）
2. 3-5トピック選定
3. 各トピック → 5プラットフォーム用テキスト生成
4. 一括投稿

### 日次（夜: 10-15分）
1. 音声で日報入力（やったこと、学び、考え）
2. プラットフォーム別に整形
3. 投稿

### 週次（30分）
1. 週のSNS投稿から記事化候補を選定（エンゲージメント順）
2. 1-3本の記事を起案
3. ニュースレター素材をキュレーション
4. 過去記事の陳腐化チェック（自動 or 手動）

### 月次（1時間）
1. 全プラットフォームのパフォーマンス確認
2. プレスリリースネタの棚卸し
3. コンテンツ戦略の微調整
4. 仕組みの改善ポイント洗い出し

## ヒアリング項目（Phase 1）

ループ設計時に確認すべきこと:

1. **事業の種類**: 何をやっている会社か
2. **日常のインプット**: 何に日々触れているか（業界ニュース、開発、顧客）
3. **現在のアウトプット**: 今どこに何を出しているか
4. **使える時間**: 1日何分コンテンツに使えるか
5. **目的**: 認知拡大/リード獲得/採用/ブランディング
6. **強みのコンテンツ**: 何なら語れるか（技術/事例/意見/ニュース）
7. **既存資産**: 過去にやったけどコンテンツ化してないもの
